# Portfolio Roadmap

## Current State — Cycle 49: Completion-First Active Development (2026 Q2)

All 13 repositories are already live, indexed, and under active iteration. The portfolio no longer follows a one-time bootstrap model; it now evolves in numbered cycles, with each cycle selecting one repository for a concrete improvement, test validation, documentation sync, and memory update. `phishing-surface-monitor` has now closed its full v0.2 Certificate Transparency block (stream monitoring, wildcard alerts, and new-certificate notifications), while `honeypot-foundry` is the next completion-first target and `dfir-attack-lab` stays in beyond-ready maintenance mode.

## Continuous Cycle Model

1. Review `MEMORY.md` and the `README.md` Evolution Log
2. Choose the next repository in the rotation
3. Run or repair the local install/test baseline
4. Ship one meaningful defensive improvement
5. Update documentation, license references, and central indexing
6. Record the cycle summary and queue the next target

## 2026 Priorities

- Close remaining roadmap gaps inside the 13 permanent repositories
- Keep CLI entrypoints, editable installs, and test suites healthy across the portfolio
- Expand protocol coverage, exporters, dashboards, and deployment examples where still missing
- Improve cross-repository interoperability between telemetry, DFIR, IR, WAF, cloud, IAM, secrets, and AI security tooling

## 2027 Direction

- Promote the strongest repositories from **Active Development** to **Operational**
- Publish reference deployments, case studies, and more guided learning paths
- Support community contributions while preserving the 13-repository structure

## Governance

The 13-repo structure is permanent. Growth happens through depth, not breadth.
